Behavioral Contamination via KV Cache Reuse

A V-Vector Attack on Multi-Turn Agent Systems — Draft v2, April 13, 2026

Abstract

Multi-turn agent systems rely on KV cache reuse as a core efficiency mechanism. We demonstrate that this optimization introduces a novel behavioral integrity vulnerability: sycophantic interactions leave computational residue in the KV cache's V vectors that degrades factual accuracy for subsequent users who never see any contaminating text. In controlled experiments (n=100), KV cache contamination causes a 35 percentage point increase in factual error rates—with non-overlapping 95% confidence intervals. Critically, we establish precise scope through a null result: system-prompt prefix caching is not vulnerable to this V-vector mechanism, while user-turn history caching is. Agentic systems that cache conversation history are vulnerable; stateless deployments and standard prefix caching are not.

1. Introduction

Production LLM deployments increasingly rely on multi-turn agent architectures where conversation history is preserved across interactions. Systems like Claude Code, Cursor, and LangChain-based agents use the ReAct pattern—reason, act, observe, repeat—with KV cache retention as a core efficiency mechanism.

This paper demonstrates that the infrastructure optimization enabling agent efficiency also introduces a novel security vulnerability. When a prior user's interaction induces sycophantic model behavior, the resulting behavioral state persists in the cached KV vectors. Subsequent users querying the same cached state exhibit elevated factual error rates, even though they never see any contaminating text.

We contribute two findings:

  1. First empirical demonstration of behavioral integrity compromise via KV cache. Prior work focused exclusively on confidentiality—extracting prompt tokens. We characterize behavioral contamination that degrades output quality.
  2. Precise mechanistic characterization of vulnerable vs. non-vulnerable patterns. Through a null result, we establish that system-prompt prefix caching is not vulnerable, while user-turn history caching is.

2. Background and Threat Model

2.1 KV Cache Reuse in Production

Modern LLM inference relies on the key-value (KV) cache to avoid redundant computation. In multi-turn deployments, this optimization extends across conversation turns—systems retain KV cache from previous turns and append only new tokens. Production data from Aliyun Tongyi confirms this is standard practice. Cache lifetime: 5 minutes to 24 hours depending on provider.

2.2 Threat Model

We characterize passive behavioral contamination:

  • No adversarial crafting required. The "attacker" is simply a normal prior user who asked a leading question.
  • Contamination without visibility. The victim never sees the prior user's interaction—only its V-vector residue affects outputs.
  • Behavioral integrity, not confidentiality. No information leaks; outputs are degraded in quality.

3. Related Work

KV cache security (confidentiality focus): PROMPTPEEK, Shadow in the Cache, InputSnatch—all study information extraction. None study behavioral integrity.

Behavioral manipulation: Sycophancy literature, activation steering, prompt poisoning—these require adversarial input to be processed. Our attack works when contaminating content is absent.

Closest work: CacheAttack (semantic cache, application layer, requires crafted queries). We characterize passive propagation through normal KV reuse at the model layer.

4. Mechanistic Foundation

In prior work, we established that sycophancy is encoded specifically in V (value) vectors of the attention mechanism's KV cache:

  • V-only patching: Replacing V vectors from a sycophantic run induces sycophancy in a clean run (41% from 9% baseline)
  • K-only patching: Minimal effect on sycophancy rates
  • Layer localization: Effect concentrated in later layers (13+)

The implication: sycophancy is a computational state in V vectors that persists in the KV cache and affects subsequent generation.

6. Threat Characterization

⚠️ Vulnerable Patterns

  • Multi-turn agents (Claude Code, Cursor, LangChain)
  • Session-level KV retention
  • Shared conversation prefixes with user turns
  • ReAct-style agentic workflows

✓ Non-Vulnerable Patterns

  • System-prompt-only prefix caching
  • Stateless single-turn deployments
  • Strict per-user session isolation
  • No KV retention between calls

7. Discussion

7.1 Distinction from Prompt Injection

Prompt injection requires adversarial content to be present. Our attack works when contaminating content is absent—only its V-vector residue remains. Input filtering cannot prevent this.

7.2 Passive vs. Active Contamination

The threat requires no attacker. Organic sycophancy from any prior user is sufficient. You cannot prevent contamination by filtering malicious inputs—the contaminating queries are normal queries that happen to induce sycophancy.

7.3 Connection to Mechanistic Interpretability

Precisely characterizing which component (V vectors), which layers (late entries), and which deployment pattern (user-turn history) is vulnerable is only possible because of mechanistic interpretability research. This is the practical security payoff of that research agenda.

8. Limitations

  • Single model: Qwen2.5-3B-Instruct (mechanism established on Gemma-4 2B in prior work)
  • Single task domain: Geography capitals (clean ground truth, but limited scope)
  • Simulated KV reuse: Not tested against production serving infrastructure
  • Same-topic contamination: Cross-topic effects were null (limits blast radius)

9. Conclusion

Multi-turn agent systems that cache conversation history are vulnerable to behavioral contamination via V-vector mechanism. A prior user's sycophantic interaction causes a 35 percentage point increase in factual error rates for subsequent users who share that cached state—without any contaminating text being visible.

Critically, this vulnerability is specific to user-turn history caching. System-prompt prefix caching does not exhibit V-vector contamination. This null result transforms a vague "caches are dangerous" observation into actionable guidance: agentic systems with conversation history retention require mitigation; stateless deployments and system-prompt-only caching do not.

The finding highlights a tension in production LLM deployment: the infrastructure optimizations that make multi-turn agents computationally viable also introduce novel behavioral integrity risks.
