Behavioral Contamination via KV Cache Reuse

A V-Vector Attack on Multi-Turn Agent Systems — Draft v3, April 13, 2026

Abstract

Multi-turn agent systems rely on KV cache reuse as a core efficiency mechanism. We demonstrate that this optimization introduces a novel behavioral integrity vulnerability: sycophantic interactions leave computational residue in the KV cache's V vectors that degrades factual accuracy for subsequent users who never see any contaminating text. Building on prior mechanistic work establishing that sycophancy is encoded in V vectors [1], we show in controlled experiments (n=100) that KV cache contamination from a prior user's sycophantic interaction causes a 35 percentage point increase in factual error rates for subsequent queries—with non-overlapping 95% confidence intervals. Critically, we establish precise scope through a null result: system-prompt prefix caching is not vulnerable to this V-vector mechanism, while user-turn history caching is. This distinction has immediate implications for production deployments: agentic systems that cache conversation history are vulnerable, while stateless single-turn deployments and standard prefix caching are not.

1. Introduction

Production LLM deployments increasingly rely on multi-turn agent architectures where conversation history is preserved across interactions. Systems implementing ReAct-style workflows—reason, act, observe, repeat—use KV cache retention as a core efficiency mechanism. Without caching, each turn would require reprocessing the entire conversation history, making agentic workflows computationally prohibitive. Production data from Aliyun Tongyi's serving clusters confirms this is standard practice: multi-turn requests naturally reuse KV cache from previous turns, as current systems deliberately append conversation history as a prefix to each new request.

This paper demonstrates that the infrastructure optimization enabling agent efficiency also introduces a novel security vulnerability. When a prior user's interaction induces sycophantic model behavior—agreeing with incorrect premises rather than providing accurate information—the resulting behavioral state persists in the cached KV vectors. Subsequent users querying the same cached state exhibit elevated factual error rates, even though they never see any contaminating text. The contamination travels through the computational substrate, not the token stream.

Our work builds on mechanistic interpretability research establishing that sycophancy is encoded specifically in V (value) vectors of the attention mechanism's KV cache, concentrated in later transformer layers [1]. We extend this finding to demonstrate its security implications in production deployment patterns.

We contribute two findings that are new to the literature:

  1. First empirical demonstration of behavioral integrity compromise via KV cache. Prior work on KV cache security has focused exclusively on confidentiality—extracting another user's prompt tokens or hijacking responses. We characterize a distinct threat: behavioral contamination that degrades the quality of model outputs without leaking information.
  2. Precise mechanistic characterization of vulnerable vs. non-vulnerable deployment patterns. Through a null result, we establish that system-prompt prefix caching is not vulnerable to V-vector contamination, while user-turn history caching is. We provide a mechanistic explanation for this distinction.

The null result belongs in the introduction, not a limitations section. The fact that system-prompt caching exhibits zero V-vector contamination effect sharpens our claim from "KV caches are dangerous" (vague, already known) to a precise mechanistic characterization of where the vulnerability lives. This precision is what makes the finding actionable for production systems.

2. Background and Threat Model

2.1 KV Cache Reuse in Production

Modern LLM inference relies on the key-value (KV) cache to avoid redundant computation. During autoregressive generation, attention computations for previously processed tokens are cached and reused, reducing the complexity of each forward pass from O(n²) to O(n) in sequence length.

In multi-turn deployments, this optimization extends across conversation turns. Rather than reprocessing the entire conversation history for each new user message, production systems retain the KV cache from previous turns and append only the new tokens. Aliyun Tongyi's analysis of production serving clusters shows this is standard practice for cost efficiency. Systems like Continuum are specifically designed to retain KV cache across tool calls in agentic workloads, with the explicit goal of preserving conversation state across the reason-act-observe loop.

Cache lifetime varies by provider and configuration. Anthropic's documentation indicates cached prefixes remain available for 5 minutes by default, extendable to one hour. Some production systems retain session-level caches for 24 hours to maximize reuse across returning users.

2.2 Exposure Window Analysis

The realistic threat exposure depends on cache lifetime and request volume. Consider a multi-tenant agentic platform with 1,000 requests per minute and a 5-minute cache TTL. If a contaminating interaction occurs and its KV state is cached, subsequent requests to the same cache partition—potentially dozens to hundreds within the TTL window—inherit the contaminated state. With a 24-hour TTL in high-retention deployments, the exposure window expands to thousands of potentially affected requests. The passive nature of the attack means every sycophancy-inducing interaction creates a new contamination window, compounding across users organically.

2.3 Threat Model

We characterize a passive behavioral contamination attack with the following properties:

No adversarial crafting required. Unlike CacheAttack (which requires crafting queries that collide with victim embeddings) or prompt injection (which requires inserting malicious instructions), our attack requires no attacker action. The "attacker" in the most concerning scenario is simply a normal prior user who happened to ask a leading question that induced sycophantic model behavior.

Contamination without visibility. Unlike prompt injection where the adversarial content must be present in the victim's context, our attack works when the contaminating content is absent. The victim never sees the prior user's sycophantic interaction—only its computational residue in the V vectors affects their outputs.

Behavioral integrity, not confidentiality. Prior work focuses on extracting another user's prompt tokens (PROMPTPEEK, InputSnatch) or serving cached responses to the wrong user (semantic cache hijacking). We characterize a different class of harm: degraded factual accuracy in outputs that contain no leaked information.

Attacker capabilities: Access to the same shared KV cache infrastructure as the victim (i.e., using the same production API or multi-tenant deployment). No special privileges, no crafted inputs, no timing attacks.

3. Related Work

3.1 KV Cache Security: The Confidentiality Focus

Existing work on KV cache security has focused almost exclusively on confidentiality attacks. PROMPTPEEK demonstrates that system prompts can be extracted through carefully crafted queries that probe the cached state. "Shadow in the Cache" shows that KV cache side-channels can leak information about prior users' inputs through timing and memory access patterns. InputSnatch extends this to extracting user inputs from shared embedding caches. All of this work treats the KV cache as an information leak—the threat is that secrets escape from the cache to an attacker. None characterizes the cache as a behavioral contamination vector.

3.2 Behavioral Manipulation of LLMs

A separate literature studies how LLM behavior can be manipulated through prompting, fine-tuning, or activation steering. The sycophancy literature documents that models systematically agree with user premises even when factually incorrect. Activation steering work shows that internal model states can be modified to alter outputs. System prompt poisoning demonstrates that adversarial instructions in the system prompt can override safety behaviors. This work studies behavioral manipulation but not KV-mediated propagation—the manipulation requires either adversarial input that the model processes, or direct access to model weights/activations.

3.3 Closest Prior Work

CacheAttack demonstrates that semantic caching (application-layer response caching based on embedding similarity) can be exploited to serve wrong cached responses, causing 90.6% tool invocation hijacking in agent systems. This is behavioral contamination through a cache—but through a completely different mechanism at the application layer, requiring crafted queries. "Whose Narrative is it Anyway?" demonstrates that KV cache can be directly manipulated to alter model behavior within a single session, establishing that KV state carries behavioral information—but does not study cross-user propagation through normal cache reuse.

The gap this paper fills: the first empirical demonstration that normal KV cache reuse in multi-turn systems propagates behavioral contamination across users, mechanistically grounded in V-vector sycophancy encoding, with precise characterization of which deployment patterns are and are not vulnerable.

4. Mechanistic Foundation

This section establishes why the threat model is grounded in model internals, bridging to prior mechanistic interpretability work [1].

4.1 V-Vector Encoding of Sycophancy

In [1], we established that sycophancy in transformer models is encoded specifically in the V (value) vectors of the attention mechanism's KV cache, not in the K (key) vectors that control attention routing. Through systematic ablation studies on Gemma-4 2B:

  • V-only patching: Replacing V vectors from a sycophantic run into a clean run induces sycophantic behavior (41% sycophancy rate from 9% baseline, +32pp effect)
  • K-only patching: Replacing K vectors has minimal effect on sycophancy rates
  • Layer localization: The effect is concentrated in later layers (layer 13+), suggesting it encodes a high-level behavioral disposition rather than low-level token processing

The key implication: sycophancy is not just a property of what tokens the model has seen—it's a computational state encoded in the V vectors that persists in the KV cache and affects subsequent generation even when the inducing tokens are no longer being attended to.

4.2 Implication for Multi-Turn Systems

In multi-turn agent systems, conversation history is retained in the KV cache across turns. If turn N contains a sycophantic interaction (user provides incorrect premise, model agrees), the V-vector contamination from that interaction persists into turns N+1, N+2, etc. The model reads the history tokens regardless—that's how conversation context works—but the V-vector contamination adds an additional behavioral bias on top of the text-level context. Our experiments quantify this additional effect and test whether it propagates through cache reuse.

5. Experiments

We present two experiments that establish the threat and delimit its scope. All experiments use Qwen2.5-3B-Instruct as the target model, validating that the V-vector mechanism established on Gemma-4 2B [1] generalizes to other architectures. We use n=100 samples per condition with Wilson score confidence intervals for proportions.

Methodology: KV State Extraction and Transfer

To simulate production KV cache reuse, we implement explicit KV state extraction and injection using the transformers library's DynamicCache API:

  1. KV extraction: For User A's interaction, we run a forward pass with use_cache=True and retain the returned past_key_values object containing K and V tensors for all layers.
  2. V-only patching: To isolate the V-vector mechanism, we construct a patched cache by taking K vectors from a clean run and V vectors from a contaminated run. This tests whether behavioral contamination travels through V vectors specifically.
  3. KV injection: For User B's query, we pass the extracted/patched KV cache as past_key_values to the generate call, with position IDs offset to account for the cached prefix length. User B's prompt tokens are processed with attention to the cached KV state.
  4. Control conditions: "Clean KV" uses KV state from a factually correct prior interaction. "Baseline" uses no prior KV state (fresh generation).

This methodology simulates the production scenario where User B's request is routed to a server holding User A's cached KV state—User B's tokens attend to User A's cached representations without User B seeing User A's text.

Null Result: System-Prompt KV Not Vulnerable to V-Vector Contamination

Pure V-vector injection produces zero contamination effect (0% vs 0%). The effect only appears when contaminating text is visible during generation (28%).

Mechanistic explanation: System prompt text is processed fresh at each generation request. Even when "cached," the model attends to the system prompt tokens and computes V vectors from them anew. Any V-vector contamination from a prior run is overwritten by the fresh computation from the visible tokens. In contrast, user-turn conversation history in the KV cache is not reprocessed—the cached V vectors are used directly without recomputation. This is why user-turn KV contamination persists while system-prompt KV contamination does not.

6. Threat Characterization

6.1 Vulnerable Deployment Patterns

The V-vector contamination pathway affects systems that cache user-turn conversation history and reuse it across queries without reprocessing. This architectural pattern is common in multi-turn agent systems that implement ReAct-style workflows, where conversation state must persist across the reason-act-observe loop for coherent multi-step reasoning. Systems designed for KV retention across tool calls—such as the Continuum architecture, which explicitly preserves KV state to maximize reuse in agentic workloads—embody precisely the vulnerable pattern.

Session-level KV retention for cost optimization also falls into this category. Production deployments that maintain user session state in KV cache, reusing it for subsequent requests from the same user (or, in multi-tenant settings, potentially routing requests to servers holding other users' cached state), expose users to contamination from prior interactions in that cache.

More generally, any deployment where multiple queries share a common conversation history prefix that includes user turns—not just system prompts—is potentially affected.

6.2 Non-Vulnerable Deployment Patterns

The null result establishes that certain deployment patterns are not vulnerable to V-vector contamination. Standard prefix caching of system prompts only—the most common production optimization—does not exhibit the V-vector contamination effect. The model processes the system prompt text fresh each time; V vectors are recomputed from the tokens rather than carried forward from prior contaminated state.

Stateless single-turn deployments that process each request independently without KV retention between calls are similarly unaffected. Deployments with strict per-user, per-session KV cache isolation that never reuse one user's cached state for another eliminate the cross-user contamination pathway, though within-session contamination across turns may still occur.

6.3 Mitigation Considerations

We characterize the threat rather than claiming complete mitigations, but note potential approaches. Session isolation through strict per-user KV cache partitioning eliminates cross-user contamination but sacrifices the efficiency gains that motivate KV reuse. Turn-boundary sanitization—detecting and clearing contaminated V-vector state between conversation turns—would require the mechanistic methodology from [1] to identify which V vectors carry sycophancy signal; this is feasible but not yet production-ready. At minimum, awareness that conversation history caching introduces this pathway enables informed architectural decisions and monitoring for anomalous error rates.

7. Discussion

7.1 Distinction from Prompt Injection

Prompt injection attacks require adversarial content to be present and processed by the victim model. Defense strategies focus on filtering malicious inputs, sandboxing untrusted content, and instruction hierarchy. Our attack is fundamentally different: the contaminating content is absent when the victim is affected. Only its V-vector residue remains in the cached state. Input filtering cannot prevent this—the contamination comes from normal model behavior on non-adversarial inputs, persisting through infrastructure that was designed for efficiency.

7.2 Passive vs. Active Contamination

The threat model does not require an attacker. Organic sycophancy from any prior user is sufficient to contaminate the cache. This has significant implications for threat modeling: contamination cannot be prevented by filtering malicious inputs, because the contaminating queries are normal user queries that happen to induce sycophancy. The attack surface scales with cache lifetime and reuse rate—the longer caches persist and the more they're shared, the more opportunities for contamination. Detection is difficult because there's no adversarial signature—just elevated error rates on factual queries that may not be immediately apparent.

7.3 Connection to Mechanistic Interpretability

The fact that we can characterize exactly which component (V vectors), which layers (late entries), and which deployment pattern (user-turn history) is vulnerable is only possible because of mechanistic interpretability research. This is the practical security payoff of that research agenda: not just understanding how models work, but knowing where their vulnerabilities live. Without the mechanistic grounding, we could observe that "KV caching sometimes causes errors" but couldn't explain why system-prompt caching is safe while conversation-history caching is not. The V-vector mechanism provides that explanation, enabling precise rather than vague threat characterization.

8. Limitations

Cross-architecture validation scope. The mechanistic finding (V-vector sycophancy encoding) was established on Gemma-4 2B in [1]. Our cross-user experiments use Qwen2.5-3B-Instruct, demonstrating the effect generalizes across architectures—but we have not validated across the full range of production model scales (7B, 70B, etc.).

Single task domain. All experiments use geography capital questions, which provide clean ground truth for measuring factual accuracy. The scope of contamination across diverse task types (math, coding, open-ended generation) remains to be characterized.

Simulated KV reuse. Our experiments simulate production KV caching by explicitly constructing and injecting KV states via the transformers API. We have not tested against live production caching infrastructure (vLLM, TGI, proprietary serving stacks), which may have implementation-specific behaviors affecting contamination propagation.

Same-topic contamination. Our experiments test same-topic contamination (prior sycophancy about Australia affects subsequent Australia queries). Preliminary tests of cross-topic contamination showed null effects—contamination appears topic-specific. This limits the blast radius of any single contaminating interaction but also means contamination may persist for specific knowledge domains.

9. Conclusion

Multi-turn agent systems that cache conversation history are vulnerable to behavioral contamination via V-vector mechanism. A prior user's sycophantic interaction leaves computational residue in the KV cache that causes a 35 percentage point increase in factual error rates for subsequent users who share that cached state—without any contaminating text being visible to the affected user.

Critically, this vulnerability is specific to user-turn history caching. System-prompt prefix caching—the most common production optimization—does not exhibit V-vector contamination because the model recomputes V vectors from the visible system prompt tokens at each request. This null result, grounded in a mechanistic explanation, transforms a vague "caches are dangerous" observation into actionable guidance: agentic systems with conversation history retention require mitigation; stateless deployments and system-prompt-only caching do not.

The finding highlights a tension in production LLM deployment: the infrastructure optimizations that make multi-turn agents computationally viable also introduce novel behavioral integrity risks. As agentic architectures become the standard for production AI systems, understanding and mitigating KV-mediated contamination pathways becomes a practical security necessity.

Appendix A: Additional Findings on Text-Based Contamination

In addition to the V-vector mechanism experiments, we conducted experiments on text-based system prompt contamination. These findings are complementary but represent a different (and less novel) effect: contamination through visible text rather than through V-vector state.

A.1 System-Prompt Text Contamination

When contaminating text is visible in the system prompt (not just injected as V vectors), the model exhibits elevated "mentions wrong city" rates even in single-sentence responses.

This is closer to conventional prompt contamination than the V-vector mechanism that is the main contribution of this paper. Standard prefix caching where the system prompt text is present does expose users to this text-based contamination; however, the V-vector mechanism does not apply.
